# CoaLab ART ADVISORY サイト — 運用改修メモ

最終更新: 2026-07-04

## 決定事項：コンテンツ更新はスプレッドシート連携方式

代表（非エンジニア）が自分でサイトの情報を更新できるようにする。
Git CMS（Decap/Sveltia）案とも比較した結果、案件として長期運用の
保守性・引き継ぎやすさを優先し **Google スプレッドシート連携** に決定。

- サイトは静的HTMLのまま（サーバー・DB・ビルド不要）
- シートを「ウェブに公開」し、ページ表示時に JS が公開CSVを取得して描画
- シートが読めない場合は、HTMLに直書きした現行カレンダーを
  フォールバックとして表示（ページが壊れない設計にする）

### 対象コンテンツと運用分担

| コンテンツ | 更新方法 |
|---|---|
| Calendar（アートフェア年間カレンダー） | シートに1行追加（月／フェア名／都市 日英） |
| Journal（記事一覧） | 本文は note に書く。シートにタイトル・日付・noteリンクを1行追加 |

Journal の本格ブログ化（本文をサイト内で管理）はしない。
本文は note、サイトは入口という役割分担。

### ユーザー側（Googleアカウント）で必要な操作

1. Google スプレッドシートを作成（雛形はこちらで用意）
2. ファイル → 共有 → ウェブに公開（CSV形式）をオン
3. 公開URLを共有してもらう → JSに埋め込み
